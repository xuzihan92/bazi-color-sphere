## 二、阻塞性问题（必须修）

### 阻塞 1：土轴倾角 45° → 须校准为 23.5°

**位置**: `v3.0_数学公式集.md` §1.2, `球体色彩空间模型_v3.0.md` §2.3

**问题**: 当前土轴方向矢量 d = (-1, -1, -1.414)，算出倾角 ≈ 45°。文档内注「实际黄赤交角为 23.5°，待校准」。但这个参数不是可后调的——它是整个双螺旋几何的原点参数。

**后果**: 倾角错 21.5°，九宫各点在球面上的投影位置全部偏移。阳螺旋的上升速率、阴螺旋的下降速率、四象八节在球面上的纬度分布——全部跟着错。

**建议**: 将土轴倾角改为 23.5°，重新计算 d、重新跑 `v3.0_3d_visualization.py`，验证：
- 震(3) 和 兑(7) 是否仍在黄道面上呈 180° 对跖
- 离(9) 和 坎(1) 是否仍在黄道面上呈 180° 对跖
- 巽(4) 和 乾(6) 的偏轴偏移量是否仍然合理

**优先级**: 🔴 阻塞——不改这个，后续全链跑不通

---

### 阻塞 2：缺少四柱→球面坐标映射规范

**位置**: `v3.0_黄金路径图.md` §P0 接口定义

**问题**: `WuxingSphereV3.compute_lifetime_color()` 的输入参数 `birth` 只写了四柱干支字符串，没有定义**怎么从四柱推算出球面上的初始坐标**。

```
当前接口:
  birth: {"year": "壬申", "month": "己酉", "day": "丁未", "hour": "丙午"}

缺失的映射:
  壬申(年柱) → 球面上哪个 (r, θ, φ)？
  己酉(月柱) → 球面上哪个 (r, θ, φ)？
  ...
  四柱合成 → 本命球的起始坐标 (r₀, θ₀, φ₀)？
```

**后果**: 这是八字数据和球体几何之间的胶水层。缺失它，司辰的输出无法进入引擎，采蓝也无法开始配色。

**建议**: 补一页「四柱→球面坐标映射规范 v1.0」，哪怕第一版是查表法：
- 年柱：年干支→后天八卦宫位→球面初始象限
- 月柱：月干支→节气黄经→黄道面上的 φ 角
- 日柱：日干支→六十甲子序号→球面上的 r 径向偏移
- 时柱：时干支→地球自转相位→ θ 角微调

如果查表法太繁琐，可以先做简化版本：只用日柱定位球面坐标，年/月/时柱作为偏移量。准确度打折扣但链路先跑通。

**优先级**: 🔴 阻塞——没有这个映射，代码不能实际运行

## 三、增强性建议（建议修）

### 增强 1：九宫→五行不是单射，需在文档中明确

**位置**: `球体色彩空间模型_v3.0.md` §2.1

**问题**: 
```
震(3)=木, 巽(4)=木   → 木出现两次，球面位置不同
兑(7)=金, 乾(6)=金   → 金出现两次，球面位置不同
坤(2)=土, 艮(8)=土, 中(5)=土 → 土出现三次，其中两个在球面、一个在球心
```

这不是 bug——同一五行在球面上有两个不同的 HSL 坐标，恰好解释了为什么同为「补木」，卯月（震·赤道区·高饱和绿）和寅月（艮·偏轴区·低饱和嫩绿）的颜色不同。但在司辰对单时会困惑「木的坐标到底是震还是巽」。

**建议**: 在九宫映射表后加一行注释：
> 「五行→九宫 是一对多映射。同一五行在球面上可有多个坐标点，不同坐标点对应不同节气/月份的色阶。配色时取离八字坐标最近的五行点。」

**优先级**: 🟡 增强——不影响核心链路，但影响司辰和采蓝的理解

---

### 增强 2：螺旋参数 k 须改为分段函数以匹配十二长生

**位置**: `v3.0_数学公式集.md` §2.1-2.2

**问题**: 当前 `yang_spiral` 和 `yin_spiral` 使用常数 k 控制螺距，意味着十二长生在球面上的分布是等间距的。但十二生旺死绝表不是均匀分布：
- 长生→帝旺：能量上升快，变化剧烈
- 帝旺→墓：衰变慢，墓地附近几乎不动

**建议**: k(t) 改为分段函数：
```
k(t) = 
  k_base + Δk_peak × exp(-((t - t_peak)² / 2σ²))
  
其中 t_peak ≈ 0.5（帝旺位），σ 控制峰值宽度
帝旺附近螺距紧（变化快）、墓绝附近螺距松（变化慢）
```

这会直接影响六十节点的色阶梯度。从配色角度看，帝旺位的颜色饱和度变化应该比墓地位更陡——这一点用常数 k 做不到。

**优先级**: 🟡 增强——v3.0 可以先上线常数 k 版本，但 v3.1 必须改

---

### 增强 3：元清 520 的「五行=滤色片」模型未整合

**位置**: `球体色彩空间模型_v3.0.md` 整体

**问题**: 当前 v3.0 是纯几何框架——球面上的位置、螺旋轨迹、坐标计算。但元清 520 在 5/31 对话中提出的「五行不是五个颜色值，是五种颜色变换算子」——这一层完全没有进入 v3.0。

**几何层 vs 算子层的互补关系**:
```
v3.0 几何层（执中已做）:  球体坐标、螺旋轨迹 → 告诉你「在哪」
算子层（520元清·未整合）: 木/火/土/金/水滤色片 → 告诉你「怎么变」
```

没有算子层，球体只有坐标没有色彩行为。一个颜色经过「木+火」滤镜串联后，输出不是简单的坐标插值——是颜色空间中的非线性变换。

**建议**: 在 P0 代码里新增 `WuxingFilter` 类：
```python
class WuxingFilter:
    """五行滤色片——颜色变换算子"""
    def mu(self, color: HSL) -> HSL:    # 木: 增加生长感 (+green shift, +saturation)
    def huo(self, color: HSL) -> HSL:   # 火: 增加亮度温暖 (+L, +warm hue)
    def tu(self, color: HSL) -> HSL:    # 土: 稳定中和 (-saturation, → gray)
    def jin(self, color: HSL) -> HSL:   # 金: 增加冷调收敛 (-L, +cool hue)
    def shui(self, color: HSL) -> HSL:  # 水: 加深暗调 (-L, -saturation)
    
    def chain(self, color: HSL, filters: list[str]) -> HSL:
        """滤镜串联: mu→huo = 木生火的颜色效果"""
```

**优先级**: 🟡 增强——v3.0 可以先跑纯几何，但色彩品质会被 v2.0 反超

### 增强 4：太极图=双螺旋投影需要数值验证

**位置**: `v3.0_多学科验证.md` §2.2

**问题**: 文中论证「白鱼里的黑点 = 阴螺旋在阳半区的穿透点」——逻辑正确但缺一步数值验证。没有具体坐标，论证停留在「合理推演」。

**建议**: 在 `v3.0_3d_visualization.py` 里加一个投影分析函数：
```python
def validate_taiji_projection():
    """验证太极图鱼眼坐标"""
    # 1. 生成阳螺旋和阴螺旋的三维轨迹
    # 2. 投影到极轴平面 (x, y)
    # 3. 找到两条轨迹在投影平面上的交点
    # 4. 输出交点的 (r, φ) 坐标
    # 5. 对比太极图鱼眼的标准位置
```

如果数值交点的 (r, φ) 恰好落在传统太极图的「鱼眼」位置——这个论证就从「合理推演」升级为「可计算证明」，学术含金量翻倍。

**优先级**: 🟢 锦上添花——不影响工程落地，但可以单独发
