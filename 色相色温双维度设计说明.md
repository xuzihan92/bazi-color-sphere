# 色相 · 色温 双维度设计说明

> 作者: 司辰 | 日期: 2026-06-08  
> 背景: 回应元清审查意见「四行色相角映射不一致」及梓涵「色相和色温是不是应该都要有」  
> 关联: 执中 · 审查回应与修正计划

---

## 问题起源

执中 v2.0 方案中，四行采用固定 HSL 色相角映射：

```
木=120°(绿)  火=0°(红)  金=210°(白蓝)  水=270°(深蓝)
```

元清发现核心矛盾：天文上春分→夏至=90°，但色相环上绿→红=120°，线性映射不成立。

同时梓涵提出：颜色本身的冷暖（色温）和五行的冷暖（春温夏热秋凉冬寒）是两个维度，不应混同。

---

## 设计决策：色相和色温不二选一，并行

| 维度 | 字段 | 来源 | 回答的问题 |
|------|------|------|-----------|
| 色相 H | 0-360° | 天干五行 | **你是谁**——木绿火红土黄金白水黑 |
| 色温 T | Kelvin | 地支季节 | **什么温度照着你**——春温夏热秋凉冬寒 |
| 中轴 L | 0-1 | 十二长生 | 你在球面哪个纬度 |
| 半径 S | 0-1 | 旺衰 | 你有多纯/多灰 |

---

## 为什么是色温而不是色相修正

如果只用色相修正（如把绿→红的距离从120°改为90°），结果是：
- 灰色和红色之间的过渡全被压缩
- 丢失了「同一绿色有冷暖之分」的维度

**用独立的色温维度**，保留了完整的色相空间，同时增加了一层信息：

```
同一件青瓷（色相=绿）
  夏天看 → 暖光照 → 偏暖绿（生机勃勃）
  冬天看 → 冷光照 → 偏冷绿（沉稳内敛）
  
瓷没有变——是光源变了。色相不变，色温在变。
```

---

## 球体模型中的物理对应

```
固有色球面 = f(五行, H, S)         ← 静态，「你是谁」
光源       = f(太阳位置, 季节)      ← 动态，「此刻的阳光」
最终呈现   = 固有色 × 光源冷暖     ← 你看到的颜色
```

太阳在球体天球上的位置由：
1. **太阳赤纬** — 天文数据，日级精度
2. **太岁五行** — 每年附加偏移
3. **大运五行** — 每十年底色偏移

太阳赤纬本身就带色温：夏至 +23.5° → 直射 → ~3000K 暖光；冬至 −23.5° → 斜射 → ~8000K 冷光。

---

## 输出格式

每柱输出四元组 `(H, S, L, T)`，引擎端：

```
1. 球面点定位: H=经度, L=纬度, S=距轴半径
2. 固有色: H + S → 基础颜色
3. 光照: T → 光源色温
4. 混合: 固有色在色温光源下的显色
5. 可选: T→HSL 补偿（用于视觉渲染，不改变五行判断）
```

---

## 色温→HSL 补偿（视觉层）

```
T < 4000K: 暖调 → H 向红偏移, L 偏高
4000 ~ 6000K: 中性 → H 不变, L 标准
T > 6000K: 冷调 → H 向蓝偏移, L 偏低
```

**注意**: 这是视觉渲染层的补偿，不影响引擎内部的五行定性。

---

## 设计边界

- 色相和色温互不替代——不存在「用色温表达木是绿色」
- 土柱（戊己）的 H = null，不参与环面，但通过 L/S 体现
- 色温仅由地支季节决定，不受天干五行影响（木干配子水支 → 色相绿、色温寒8000K，互不矛盾）
