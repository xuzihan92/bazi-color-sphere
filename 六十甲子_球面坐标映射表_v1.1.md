## 二、色温 T —— 阴阳冷暖（新增）

### 2.1 地支→季节→色温

| 地支 | 季节 | 节气 | 色温 K | 冷暖感 |
|------|------|------|--------|--------|
| 寅 | 春 | 立春-惊蛰 | 4500K | 温升 |
| 卯 | 春 | 春分-清明 | 5000K | 温和 |
| 辰 | 春末 | 谷雨 | 5500K | 暖→热过渡 |
| 巳 | 夏 | 立夏-芒种 | 3500K | 热 |
| 午 | 夏 | 夏至-大暑 | 3000K | 极热 |
| 未 | 夏末 | 大暑-处暑 | 3800K | 热→凉过渡 |
| 申 | 秋 | 立秋-白露 | 5500K | 凉 |
| 酉 | 秋 | 秋分-霜降 | 6500K | 清冷 |
| 戌 | 秋末 | 霜降-立冬 | 7000K | 凉→寒过渡 |
| 亥 | 冬 | 立冬-大雪 | 7500K | 寒 |
| 子 | 冬 | 冬至-大寒 | 8000K | 极寒 |
| 丑 | 冬末 | 大寒-立春 | 7000K | 寒→温过渡 |

**设计逻辑**：
- 色温从冬至极寒（8000K）→ 春分温和（5000K）→ 夏至极热（3000K）→ 秋分清冷（6500K）→ 回到冬至
- 辰戌丑未四库作为季节过渡，色温取相邻季中值
- 低 K = 暖/红调，高 K = 冷/蓝调——对应阴阳消长

### 2.2 色温→HSL 补偿

色温本身是独立输出维度，但需要时可以换算为 HSL 偏移量，供视觉渲染用：

```
色温 T → HSL 偏移：
  T < 4000K：暖调 → H 向红偏移，L 偏高（暖黄光感）
  4000K ≤ T ≤ 6000K：中性 → H 不变，L 标准
  T > 6000K：冷调 → H 向蓝偏移，L 偏低（冷白光感）
```

**注意**：这是视觉补偿，不改变五行命理判断。引擎内部保留原始 H 值用于五行定性。

## 五、四柱权重

| 柱 | 权重 | 承载维度 |
|-----|------|---------|
| 年 | 15% | 祖荫根基 |
| 月 | 30% | **月令→主导色温** |
| 日 | 40% | 日元→主导色相 |
| 时 | 15% | 归宿晚年 |

**加权方式**：

```
H_final = atan2( Σ(wᵢ × sin(Hᵢ)), Σ(wᵢ × cos(Hᵢ)) )    // 圆形平均
L_final = Σ(wᵢ × Lᵢ) / Σwᵢ
S_final = Σ(wᵢ × Sᵢ) / Σwᵢ
T_final = Σ(wᵢ × Tᵢ) / Σwᵢ    // 色温线性加权
```

## 七、输出 schema（建议）

```json
{
  "bazi_sphere": {
    "pillars": {
      "year":  { "ganzhi": "壬申", "H": 270, "L": 0.20, "S": 0.70, "T": 5500 },
      "month": { "ganzhi": "己酉", "H": null, "L": 0.75, "S": 0.50, "T": 6500 },
      "day":   { "ganzhi": "丁未", "H": 8,   "L": 0.55, "S": 0.50, "T": 3800 },
      "hour":  { "ganzhi": "丙午", "H": 0,   "L": 0.50, "S": 0.85, "T": 3000 }
    },
    "weighted": {
      "H": 3.2,
      "L": 0.52,
      "S": 0.63,
      "T": 4320,
      "dominant_wuxing": "火",
      "season_bias": "偏夏暖"
    }
  }
}
```

**字段说明**：
- `H` = null 表示土五行（居中轴，不参与环面）——引擎处理为灰度融入
- `T` = 加权平均色温，4320K 偏暖（受火旺影响）
- `dominant_wuxing` 由加权 H 推导
- `season_bias` 由加权 T 推导
