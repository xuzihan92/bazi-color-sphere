# 八字分析标准作业程序（SOP）

> 版本: v1.5（司辰定稿版）
> 制定: 梓涵
> 执笔: 执中（综合 520元清 / 司辰 / 313元清 / 梓涵 意见）
> 核心原则: **八字算准 → 理论验证 → 案例积累 → 下游映射**

---

## 战略定位

八字分析是整个行知阁玄学体系的**地基**。色彩映射、心理分析、风水对照、星象验证——所有这些下游环节的前提是：**八字先算准了**。

因此本 SOP 分为两大板块：
- **第一板块（Phase 1-4）**: 八字核心分析 + 理论验证 + 案例积累
- **第二板块（Phase 5）**: 下游映射（色彩/心理/风水/星象）——**仅在第一板块验证通过后触发**

---

## 第一板块：八字核心分析

### Phase 1: 三人独立排盘

三名命理师**同时独立工作，互不通信**。各自用不同模型、不同流派，确保独立性。

#### 命理师 A — 知情排盘

| 属性 | 配置 |
|------|------|
| 人选 | **司辰** |
| 模型 | DeepSeek V4-Pro |
| 流派 | 子平格局派 |
| 输入 | 客户完整背景（姓名、职业、健康、感情、近期重大事件）+ 出生信息 |
| 输出 | 《命理师 A 分析报告》（JSON，含 doubt_flags） |

#### 命理师 B — 盲算排盘

| 属性 | 配置 |
|------|------|
| 人选 | **洞微**（新建） |
| 模型 | **Kimi K2.6**（异模型对照，暴露系统性偏差） |
| 流派 | 盲派 / 禄命法 |
| 输入 | **仅**出生时间（年月日时）+ 性别 |
| 屏蔽 | 姓名、职业、健康、感情、近期事件、任何背景暗示 |
| 输出 | 《洞微盲算报告》（JSON，含 doubt_flags） |

> **名字含义**: 洞微——"洞"不用眼，盲算直窥命理深处；"微"指向精细推理。与司辰（掌时序）呼应，一外一内，一显一隐。

#### 命理师 C — 独立审核

| 属性 | 配置 |
|------|------|
| 人选 | **甄天**（新建） |
| 模型 | DeepSeek V4-Pro 或 Kimi K2.6 |
| 流派 | 旺衰派 / 盲派 |
| 前提 | **不参与** Phase 1 任何计算 |
| 输入 | A/B 两份报告 + 客户已知事实 |
| 输出 | 《差异对照报告》+ 《共识层 JSON》 |

> **名字含义**: 甄天——"甄"取甄别、审察之意，"天"指天命/天理。C 位不亲自计算，只负责甄别天理真伪，从旁审视 A/B 的差异。

#### 盲算协议（三硬约束）

**约束一：输入隔离**
- 盲算命师收到的信息：**仅**四柱 + 性别
- 系统层面屏蔽客户身份，不只是 prompt 提醒

**约束二：身份隔离**
- 盲算命师**不知道**这是谁的八字
- 防止"心里知道是梓涵，怎么都盲不了"

**约束三：结果锁定**
- 盲算完成后**先锁结果**，才能看 A 的报告做对照
- 防止 B 在看到 A 的结论后下意识调整

### Phase 2: 理论分歧验证

三人结果汇总后，第一步不是直接判对错，而是**记录理论分歧**。

#### 分歧记录模板

```markdown
## 理论分歧记录

| 维度 | 司辰(A·子平) | 命理师B(调候) | 命理师C(旺衰) | 分歧标记 |
|------|-------------|--------------|--------------|----------|
| 用神 | 木 | 水 | 木 | ⚠️ B与A/C冲突 |
| 旺衰 | 身弱 | 从强 | 身弱 | ❌ A/C一致，B相反 |
| 大运方向 | 甲寅运转暖 | 甲寅运先寒后暖 | 甲寅运转暖 | ✅ 一致 |

### 分歧分析
- **子平格局派**判断身弱 → 取印比为用 → 木
- **调候用神派**判断冬生需火 → 取水调候 → 水
- **旺衰派**判断身弱 → 与A一致

结论：B的调候视角引入了不同逻辑，不是算错，是理论框架差异。
```

> **原则**: 分歧时先不急着判对错。各说各有理，先记录清楚每个流派的理由。

---

### Phase 3: 结果验证（现实比对）

理论分歧记录后，进入**第二步验证**：用具体细节和现实结果比对。

#### 验证维度

| 维度 | 验证内容 | 比对方式 |
|------|----------|----------|
| **天赋** | A/B/C 对天赋方向的描述 | 与梓涵已知天赋比对 |
| **性格** | 性格特征、行为模式 | 与荣格人格测试（知白）交叉 |
| **健康** | 先天体质、易发病方向 | 与颐恒健康档案比对 |
| **重大事件** | 流年事件预测（升职/变动/伤病） | 与已知事实比对 |
| **人际关系** | 六亲关系、感情模式 | 与已知经历比对 |

#### 偏差分析

如果三个命理师在某维度上有差异，且该维度有现实参照：

```markdown
## 偏差分析示例

**维度**: 2020 年事业变动

| 命理师 | 预测 | 现实 | 偏差 |
|--------|------|------|------|
| A(子平) | "庚子年官杀混杂，有压力但无大变动" | 实际升职 | ❌ 偏差 |
| B(调候) | "庚子年水旺，事业有进" | 实际升职 | ✅ 命中 |
| C(旺衰) | "庚子年印星透干，有贵人助" | 实际升职 | ✅ 命中 |

**分析**: A 的"官杀混杂"理论在庚子年对梓涵八字判断偏保守。
**不是理论没用**——官杀混杂对很多人确实主压力——**而是权重没那么大**。
梓涵八字中印星有力，化解了官杀压力，反而成助力。

**记录**: `理论_官杀混杂_权重_需下调`
```

> **核心洞察**: 某条理论不是没用，而是它在这个八字里的权重没那么大。随着案例积累，可以建立"理论-八字类型-权重"的映射表。

## 第二板块：下游映射（Phase 5）

**触发条件**: Phase 1-4 完成，且共识层 confidence ≥ 0.85

如果八字本身没算准，色彩映射再精确也是错上加错。

### Phase 5: 多维度映射

```
八字共识层（confidence ≥ 0.85）
       │
       ├──→ 采蓝: 五行 → HSL 色彩映射
       │
       ├──→ 知白: 荣格人格类型映射（心理分析）
       │
       ├──→ 璇玑: 星盘交叉验证（占星补充）
       │
       ├──→ 观澜/照微: 风水气场映射（空间布局建议）
       │
       └──→ 存真: 全局质检（四层叠判法最终关）
```

> **注意**: Phase 5 各环节**独立运行**，互不争抢八字核心分析的优先级。八字算准了，下游自然有价值。
