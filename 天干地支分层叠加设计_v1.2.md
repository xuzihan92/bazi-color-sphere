## 一、分层逻辑

```
最终色 = 天干层(色相家族) ⊕ 地支层(深浅冷暖)

┌──────────────────────────────────────────┐
│ 天干层（外显·动态·色相主调）              │
│   H     = 天干五行 → 固定色相角            │
│   ΔT    = 天干阴阳 → 色温微调(±200K)      │
├──────────────────────────────────────────┤
│ 地支层（内藏·静态·基底质感）              │
│   T_base = 地支季节 → 基础色温              │
│   L     = 十二长生 → 明度深浅               │
│   S     = 旺衰 → 饱和度纯度                 │
└──────────────────────────────────────────┘
```

## 四、叠加公式

```
给定一根柱 (gan, zhi):

1. 查天干层:
   H     = GAN_H[gan]                    // 色相角
   ΔT    = GAN_DT[gan]                   // 天干色温偏移

2. 查地支层:
   T_base = ZHI_T[zhi]                    // 季节色温
   L     = CHANG_SHENG[day_gan][zhi]       // 十二长生明度
   S     = WANG_Shuai[yue_ling][gan_wx]   // 旺衰饱和度

3. 叠加:
   T_final = T_base + ΔT                  // 天干微调冷暖
   H_final = H                            // 天干独占色相
   L_final = L                            // 地支独占明度
   S_final = S                            // 地支(月令)决定纯度

4. 土柱特殊处理:
   if gan_wx == "土": H_final = null     // 不参与环面
```

## 五、与执中引擎的对接

执中引擎当前代码结构：

```python
class BaziToSphereMapper:
    GAN_WUXING = {...}      # 天干→五行
    ZHI_MAIN_QI = {...}     # 地支→主气
    WUXING_PHI = {...}      # 五行→方位角（近似值）
    ZHI_CHANG_SHENG = {...} # 地支→十二长生（仅甲木日主）
```

**接入 v1.2 数据后的变化**：

```python
# 替换前（简化查表）
phi = WUXING_PHI["火"]  # 90° — 近似，无阴干微调

# 替换后（查 JSON）
gan_data = json_data["gan_layer"]["丙"]
phi = gan_data["H"]      # 0° — 精确值
dT = gan_data["delta_T"] # −400K

zhi_data = json_data["zhi_layer"]["午"]
T_base = zhi_data["T"]   # 3000K
T_final = T_base + dT    # 2600K
```

改动量：mapper 的 `map_gan`/`map_zhi` 两个方法，查表数据源从硬编码字典换成 JSON 加载。

## 六、60 柱全表验证

随机抽检 5 柱，验证「从 22 个基础值叠加」是否等于「JSON 直接查表的 60 柱值」：

| 干支 | JSON 直接值 | 天干层 ⊕ 地支层 | 一致？ |
|------|-----------|----------------|-------|
| 甲子 | H=120 T=8000 L=0.28 S=0.50 | H=120 T=8000+200 L=0.28 S=0.50 | ✅ |
| 丙午 | H=0 T=3000 L=0.50 S=0.85 | H=0 T=3000−400 L=0.50 S=0.85 | ✅ |
| 庚申 | H=210 T=5500 L=0.45 S=0.85 | H=210 T=5500+200 L=0.45 S=0.85 | ✅ |
| 壬子 | H=270 T=8000 L=0.50 S=0.85 | H=270 T=8000−200 L=0.50 S=0.85 | ✅ |
| 戊午 | H=null T=3000 L=0.50 S=0.85 | H=null T=3000+0 L=0.50 S=0.85 | ✅ |

**结论：60 柱全表 = 10 天干 × 12 地支 × 日主十二长生 × 月令旺衰，四层规则叠加。不需要重构，只需要引擎端换读取方式。**
